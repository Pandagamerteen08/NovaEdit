NovaEdit Code of Conduct

Our Commitment

The NovaEdit community is dedicated to providing a welcoming, respectful, and inclusive environment for everyone who participates in the project.

This includes:

* Developers
* Contributors
* Testers
* Plugin creators
* Theme creators
* Documentation contributors
* Discord community members
* GitHub users
* Users providing feedback

NovaEdit exists because people from different backgrounds, skill levels, and experiences come together to build something useful. Everyone should feel comfortable participating, learning, and contributing.

⸻

Expected Behavior

Community members are expected to:

* Treat others with respect and professionalism.
* Communicate clearly and constructively.
* Be patient with beginners and new contributors.
* Accept feedback and criticism respectfully.
* Provide helpful suggestions instead of unnecessary negativity.
* Respect different opinions and approaches.
* Follow project rules and contribution guidelines.
* Give proper credit when using someone else’s work.
* Report problems responsibly.

Examples of positive behavior include:

* Helping someone understand an issue.
* Providing useful code reviews.
* Explaining mistakes respectfully.
* Sharing knowledge with newer developers.
* Encouraging improvement.

⸻

Unacceptable Behavior

The following behaviors are not acceptable within the NovaEdit community:

Harassment

Examples include:

* Personal attacks
* Threats
* Bullying
* Intimidation
* Repeated unwanted contact
* Targeted harassment

Discrimination

Discrimination based on personal characteristics or identity will not be tolerated.

Everyone should be treated with dignity and respect.

Toxic Communication

Examples include:

* Insults
* Hostile arguments
* Deliberately provoking others
* Mocking beginners
* Dismissing questions without explanation

Constructive disagreement is allowed.

Disrespectful behavior is not.

Malicious Actions

The following are prohibited:

* Deliberately introducing harmful code
* Submitting malicious plugins
* Attempting to compromise user security
* Spreading malware
* Abusing project resources
* Misrepresenting contributions

⸻

Development Standards

Contributors are expected to maintain a professional development environment.

This includes:

* Writing understandable code.
* Following project coding standards.
* Testing changes before submission.
* Documenting important changes.
* Respecting project maintainers and reviewers.

Code reviews are intended to improve the project, not criticize individuals.

⸻

AI-Assisted Contributions

NovaEdit allows the use of AI-assisted development tools.

However, contributors remain responsible for their submissions.

AI-generated content must:

* Be reviewed before submission.
* Be tested properly.
* Follow project guidelines.
* Be disclosed when required by contribution rules.

AI tools should assist development, not replace responsibility.

⸻

Discord Community Guidelines

The official NovaEdit Discord community follows this Code of Conduct.

Members should:

* Use appropriate channels.
* Avoid unnecessary arguments.
* Respect moderators and community members.
* Keep discussions related to their intended topics.
* Use support channels appropriately.

Support discussions should remain helpful and constructive.

⸻

Plugin and Theme Community Guidelines

Future NovaEdit plugins and themes must follow community standards.

Creators should:

* Respect user privacy.
* Avoid malicious behavior.
* Provide accurate descriptions.
* Respect licenses and ownership.
* Maintain professional communication.

Approved plugins and themes may be subject to additional review requirements.

⸻

Reporting Problems

If you experience or witness behavior that violates this Code of Conduct, please report it.

Possible reporting methods may include:

* GitHub issues (when appropriate)
* GitHub discussions
* Official Discord moderation channels
* Private contact methods provided by NovaEdit maintainers

Reports should include:

* What happened
* When it happened
* Where it happened
* Who was involved
* Any available evidence

Reports will be reviewed as fairly and privately as possible.

⸻

Enforcement

NovaEdit maintainers have the responsibility to enforce this Code of Conduct.

Possible actions may include:

* A private warning
* Removal of inappropriate content
* Temporary restrictions
* Removal from Discord channels
* Rejection of contributions
* Temporary or permanent bans
* Blocking future participation

Actions will depend on the severity and circumstances of the situation.

⸻

False Reports

Reports should be made honestly and responsibly.

Knowingly submitting false reports, manipulating evidence, or attempting to harm another community member through false accusations may result in moderation action.

⸻

Scope

This Code of Conduct applies to:

* The NovaEdit GitHub repository
* Pull requests
* Issues
* Discussions
* Official Discord communities
* Official community spaces
* Future official NovaEdit platforms

⸻

Maintainer Responsibilities

Maintainers should:

* Lead by example.
* Treat contributors fairly.
* Review contributions objectively.
* Communicate decisions respectfully.
* Protect the health of the community.

Maintainers are responsible for helping create an environment where people can collaborate effectively.

⸻

Changes to This Code

This Code of Conduct may be updated as NovaEdit grows.

Major changes will be documented and communicated to the community.

⸻

Final Message

NovaEdit is built by people working together.

A strong community is created through patience, respect, and cooperation.

Whether you contribute code, report bugs, create themes, suggest features, or simply use the application, you are part of the NovaEdit community.

Thank you for helping make NovaEdit a better project.
